# SHL Data Insights - Tech Stack

## Architecture
SHL API → Python (fetch) → Supabase/PostgreSQL (raw) → dbt (transform) → BigQuery (analytics) → Next.js (display)

## Layers

### Data Ingestion
- Tool: Python scripts
- Purpose: Fetch raw data from SHL open API

### Raw Storage
- Tool: Supabase (PostgreSQL)
- Purpose: Store raw/operational data, serve data to Next.js

### Transformation
- Tool: dbt
- Purpose: Clean and model raw data

### Analytics / Data Warehouse
- Tool: BigQuery (Google Cloud)
- Purpose: Analytical queries and reporting

### Infrastructure as Code
- Tool: Terraform
- Purpose: Manage BigQuery datasets, tables and GCP resources

### Frontend
- Tool: Next.js
- Host: Vercel
- URL: shl.simoncarlsson.se

### CI/CD
- Tool: GitHub Actions
- Workflows:
  - ci.yml: Linting and tests on push to main
  - cd.yml: Deploy to Vercel when CI passes
  - pipeline.yml: Scheduled data pipeline (fetch + dbt)

### Orchestration
- Tool: GitHub Actions (scheduled workflows)
- Manual trigger: workflow_dispatch

## Hosting
- Frontend: Vercel (free)
- Database: Supabase (free tier)
- Data Warehouse: BigQuery (free tier)
